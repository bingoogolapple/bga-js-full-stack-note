• 页面可视区分成了 100vw 和 100vh，在 iPhone 6 上 100vw 为 375px
  • 期望在 iPhone 6 下 1rem = 100px，即 html 的 font-size = 100px，那么多少个 vw = 100px 呢？
  • x 个 vw / 100px = 100vw / 375px => x = 100vw * 100px / 375px ≈ 26.666666666666668
  • rem 布局一定要在 body 中重置 font-size 为默认值 16px，即 0.16rem
  • PS 中量取的述职是物理像素，css 中设置的是逻辑像素，需要对量取的值除以 2
  • 可以利用蓝湖获取 rem 数值，蓝湖量出来的 px 是自动除以 2 后的逻辑像素，但量出来的 rem 还是物理像素的 rem，需要自己除以 2
  • 可以利用 PxCook 获取 rem 数值，量出来的 px 是自动除以 2 后的逻辑像素，量出来的 rem 也是自动除以 2 后的逻辑 rem
  • 可以不用 rem，结合 px-to-vw 插件只用 vw 来做适配，PxCook 直接标注单位为 vw
